# 🧾 Week 1 – Video 1 (With Explanation)

## 📌 1. What is Computer Vision?

Computer Vision =

> Teaching a machine to **understand images like humans do**

### Problem:

* Computer sees: **numbers (pixels)**
* You see: **objects + meaning**

Example:

* Image = matrix like `[255, 0, 120...]`
* You = “that’s a person sitting”

👉 So the whole field is:

> Convert **numbers → meaning**

---

## 📌 2. Why this is hard

Humans are annoyingly good at vision.

You can:

* Recognize a friend in bad lighting
* Recognize objects from weird angles
* Use context automatically

Machines:

* Need thousands of examples
* Still mess up if lighting changes

Even worse:

> We don’t fully understand how *our own brain* does this 

So yeah, we’re trying to replicate something we don’t fully understand. Great plan.

---

## 📌 3. Breaking the problem (very important)

Since full understanding is too hard, we split it:

### 🔹 Low-Level Vision

Basic stuff:

* edges
* brightness
* filters

👉 Think: “raw pixel processing”

---

### 🔹 Geometry

Understanding **space and position**

* Where is object?
* How far is it?

👉 This is where depth comes in

---

### 🔹 Mid-Level Vision

Combine features:

* edges → shapes
* shapes → structures

---

### 🔹 High-Level Vision

Actual understanding:

* “this is a car”
* “this is a classroom”

---

### 🔥 Easy way to remember:

```
Pixels → Features → Structure → Meaning
```

---

## 📌 4. Two Types of Information

### 🔸 (A) Semantic = Meaning

Answers:

* What is this?

Examples:

* dog
* building
* road

👉 Used in:

* classification
* object detection
* segmentation

---

### 🔸 (B) Metric = Measurement

Answers:

* Where is it?
* How far is it?

Examples:

* distance = 5m
* object is in front/back

---

### ⚠️ Important:

* Semantic = **understanding**
* Metric = **measurement**

---

## 📌 5. Example: Semantic Vision

Imagine an image:

* sky
* building
* car

Computer labels each part.

👉 That’s **modern deep learning vision**

---

## 📌 6. Why old methods still matter

Before deep learning:

* Humans designed features manually

Now:

* Networks learn automatically

BUT:

> Old methods give **intuition + math understanding** 

Also used in:

* physics-based models
* hybrid systems

So no, you can’t ignore them just because CNNs exist.

---

## 📌 7. Depth (Metric Vision) – Core Idea

Now the interesting part.

### How do we know depth?

Using **parallax**.

---

### 🔹 What is Parallax?

When you move:

* Near objects shift a lot
* Far objects shift less

Example:

* Sit in a train:

  * nearby pole → zooms past
  * mountain → barely moves

👉 That difference = depth clue 

---

## 📌 8. Stereo Vision (like human eyes)

We have:

* Left eye
* Right eye

Each sees slightly different image.

Brain:

* compares both
* calculates depth

---

### Machine version:

* Two cameras
* Compare positions of same object
* Use math → get depth

---

### Output:

👉 **Depth Map**

* bright = near
* dark = far

---

## 📌 9. Why humans don’t need 2 cameras always

You can close one eye and still judge depth.

Why?

Because you also use:

* memory
* size of objects
* shading
* context

Machines:

* don’t have this luxury
* rely on strict methods

---

## 📌 10. Structure from Motion (SfM)

Now things get spicy.

Instead of:

* 2 fixed cameras

We use:

* 1 moving camera

---

### Process:

* Take multiple images from different positions
* Combine them
* Build 3D structure

---

### Output:

👉 **Point Cloud (3D model)**

---

### Why harder?

Because you must find:

1. What is the structure?
2. Where was the camera?

Two unknowns = more pain.

---

## 📌 11. Real-world problems (this matters)

Reality is messy:

* Cameras shake
* Images misalign
* Can’t retake data

So we use:

* rectification (fix alignment)
* post-processing

---

## 📌 12. Advanced idea

You can even:

* take images from internet
* combine them
* reconstruct 3D scene

Example:

* Taj Mahal from random tourist photos

---

# 🎯 What you MUST remember

If exam comes and your brain freezes, remember this:

### Core ideas:

* Image = numbers → need meaning
* Semantic vs Metric
* Parallax → depth
* Stereo = 2 cameras
* SfM = moving camera + 3D reconstruction

---

# 🧠 Intuition (this is your cheat code)

* Near things move more → closer
* Far things move less → farther
* Humans use experience → machines don’t
* CV = breaking a huge problem into smaller ones

# 🧾 Week 1 – Video 2

# 🔍 Semantic Computer Vision Tasks (DETAILED NOTES + EXPLANATION)

---

## 📌 1. What is “Semantic Vision”?

Semantic vision =

> Extracting **meaning** from an image

---

### 🧠 Explanation:

When you look at an image, you don’t see pixels—you see:

* objects
* actions
* context

Semantic CV tries to make machines do the same.

---

## 📌 2. Image Classification (Recognition)

### 🔹 Definition:

* Assign **one label** to an image

---

### 🔹 Example:

* Input: image of dog
* Output: “dog”

---

### 🔹 Dataset:

* **ImageNet (~1000 object classes)** 

---

### 🧠 Explanation:

Model looks at the entire image and decides:

> “What is the most important object here?”

Even if there are multiple objects:

* it picks the **dominant one**

---

### ⚠️ Limitations:

* No location info
* Can confuse similar objects
* Sensitive to noise

---

### 🔥 Important Concept: Adversarial Learning

* Tiny noise → wrong prediction
* Humans still correct

👉 Meaning:

> Model doesn’t “understand”, it just detects patterns

---

## 📌 3. Object Detection

### 🔹 Definition:

* Identify **what + where**

---

### 🔹 Output:

* Label + bounding box

---

### 🔹 Example:

* “Person → (x1, y1, x2, y2)”

---

### 🧠 Explanation:

Instead of just saying:

> “There is a person”

It says:

> “Person is HERE in the image”

---

### 🔹 Additional Output:

* Confidence score (probability)

---

### 🔥 Real Systems:

* Face detection
* Autonomous driving
* YOLO (real-time detection)

---

## 📌 4. Segmentation (Pixel-level Understanding)

### 🔹 Definition:

* Assign label to **every pixel**

---

### 🧠 Explanation:

Detection gives a box (rough idea)
Segmentation gives **exact shape**

---

### 🔹 Example:

Each pixel labeled as:

* road
* car
* person

---

### ⚠️ Key Challenge:

* Requires **manual annotation**

👉 Humans must:

* draw object boundaries
* label each pixel 

---

### 🔥 Insight:

> Models are powerful because humans did painful labeling

---

## 📌 5. Supervised vs Self-Supervised Learning

### 🔹 Supervised Learning:

* Uses labeled data
* Example:

  * image + label
  * image + segmentation mask

---

### 🔹 Problem:

* Labeling is expensive
* Not scalable

---

### 🔹 Self-Supervised Learning:

#### Idea:

> Learn using **raw data itself**

---

### 🧠 Example (Depth Learning):

1. Take image 1
2. Move camera → image 2
3. Predict:

   * depth
   * camera motion
4. Try to reconstruct image 2 from image 1 

---

### 🔥 Key Insight:

> No ground truth needed

---

### ⚠️ Hard Part:

* Designing correct **loss function**

---

## 📌 6. Video Processing vs Image Processing

### Problem:

* Treating frames independently

---

### 🧠 Explanation:

If each frame is processed separately:

* results may flicker
* no continuity

---

### Better approach:

> Use temporal information (previous frames)

---

## 📌 7. Object Tracking

### 🔹 Definition:

* Follow object across frames

---

### 🧠 Example:

* Car ID stays same across video

---

### 🔥 Difference from detection:

* Detection = find object
* Tracking = follow object

---

### ⚠️ Challenges:

* Occlusion (object hidden)
* Multiple similar objects
* Small objects (few pixels) 

---

## 📌 8. Pose Estimation

### 🔹 Definition:

* Detect body joints (skeleton)

---

### 🧠 Explanation:

Model outputs:

* head
* shoulders
* elbows
* knees

---

### 🔥 Uses:

* Sports analytics
* Motion capture
* Fitness apps

---

### Example:

* Check boxing angle
* Analyze javelin throw

---

## 📌 9. Activity Recognition

### 🔹 Definition:

* Identify **actions**

---

### Examples:

* running
* jumping
* punching

---

### 🧠 Explanation:

Requires:

* sequence of frames (video)
* not just single image

---

## 📌 10. Image Captioning

### 🔹 Definition:

* Generate sentence from image

---

### 🧠 Example:

Input → image
Output → “A man riding a bike”

---

### ⚠️ Problem:

* Can be confidently wrong

Example:

* Yellow car → “school bus” 

---

### Insight:

> Model learns patterns, not real understanding

---

## 📌 11. Geometry & Depth (Core Concept)

### Problem:

* One image → no depth

---

### Solution:

* Use two views

---

### 🧠 Explanation:

Each camera sees object differently
→ draw rays
→ intersection gives real position

---

### ⚠️ Challenge:

* Need correct **correspondence matching**

---

## 📌 12. Depth from Single Image

### Possible using:

* Deep learning
* Self-supervised methods

---

### Reality:

* Works, but less accurate

---

## 📌 13. Structure from Motion (SfM)

### 🔹 Definition:

* Build 3D model using moving camera

---

### Output:

* 3D point cloud

---

### ⚠️ Challenge:

Must estimate:

* scene structure
* camera position

---

## 📌 14. Photometric Stereo

### 🔹 Idea:

* Same camera
* Change lighting

---

### 🧠 Insight:

Shadows → give shape information

---

## 📌 15. Real-World Challenges (VERY IMPORTANT)

### 🔹 Viewpoint variation

* Same object, different angles

### 🔹 Illumination change

* Lighting differences

### 🔹 Perspective distortion

* Near looks bigger

### 🔹 Deformation

* Objects change shape

### 🔹 Occlusion

* Hidden parts

### 🔹 Background clutter

* Noise/confusion

### 🔹 Motion blur

* Fast movement → blur

### 🔹 Intra-class variation

* Same class looks different

---

# 🎯 Final Summary (Revise This Before Exam)

* Classification → what
* Detection → what + where
* Segmentation → exact pixels
* Tracking → follow over time
* Pose → body structure
* Depth → 3D understanding

---

# 🧠 Ultimate Intuition

* Machines don’t “see”
* They **learn patterns from data**
* More data + better supervision = better performance
* But real-world variations still break them

# 📌 1. Types of Machine Learning

Machine Learning is broadly divided into:

### 🔹 1. Supervised Learning

### 🔹 2. Unsupervised Learning

(plus variants in between)

---

## 📌 2. Supervised Learning

### 🔹 Definition:

> Learning using **labeled data**

---

### 🧠 How it works:

You give:

* Input → Image
* Output → Label

Example:

* Image → “mug”
* Image → “tiger”

---

### 🔹 Training Process:

1. Show many examples
2. Tell correct answer each time
3. Model learns mapping

---

### 🧠 Example:

* Show 1000 mug images → label = mug
* Show 1000 tiger images → label = tiger

After training:

* New image → model predicts label

---

### 🔥 Key Idea:

> Model learns from **examples + correct answers** 

---

### ⚠️ Limitation:

* Requires **huge labeled data**
* Humans must annotate everything

---

## 📌 3. Types within Supervised Learning

---

### 🔹 (A) Strong Supervision

* Exact labels available
* Example:

  * Image → “cat”

---

### 🔹 (B) Weakly Supervised Learning

### 🧠 Idea:

> Use **related task supervision** to solve main task

---

### Example:

* No stereo audio data
* But have image localization data

Use:

* image info → guide audio learning 

---

### 🔥 Insight:

> Not direct supervision, but **indirect guidance**

---

### 🔹 (C) Semi-Supervised Learning

### 🧠 Idea:

> Small labeled data + large unlabeled data

---

### Example:

* 1 million images
* Only 1000 labeled

Use:

* labeled → guide learning
* unlabeled → improve model

---

### 🔥 Insight:

> Reduce labeling effort

---

## 📌 4. Unsupervised Learning

### 🔹 Definition:

> Learn from data **without labels**

---

### 🧠 Goal:

* Discover structure
* Learn representation

---

## 📌 5. Classical Example: PCA

### 🔹 What PCA does:

* Finds **important directions (features)** in data

---

### 🧠 Explanation:

* Reduce dimensions
* Keep most important info

---

### ⚠️ Limitation:

* Only **linear**
* Cannot capture complex patterns

---

## 📌 6. Unsupervised Learning in Deep Learning

### 🔹 Idea:

Use **neural network to learn representation**

---

### 🧠 Example: Autoencoder

Step-by-step:

1. Input image
2. Compress (encoder)
3. Bottleneck (small representation)
4. Reconstruct image (decoder)

---

### 🔥 Key Concept:

> Bottleneck = learned representation

---

### 🧠 What it learns:

* Important features
* Ignores noise

---

### Example:

Should ignore:

* lighting changes
* background noise

Should learn:

* face identity

---

### 🔥 Insight:

> Learn **intrinsic features**, ignore unnecessary ones 

---

## 📌 7. Clustering (Unsupervised)

### Example:

* k-means clustering

---

### 🧠 Idea:

Group similar data:

* no labels needed
* based on similarity

---

## 📌 8. Traditional Computer Vision Pipeline

### 🔹 Steps:

1. Feature extraction (manual)
2. Classifier (separate)

---

### 🧠 Example:

* Extract SIFT features
* Feed to SVM classifier

---

### 🔥 Key Idea:

> Features and classifier are **separate modules**

---

## 📌 9. Handcrafted Features

### Examples:

* SIFT
* HOG
* SURF
* Harris corner

---

### 🧠 Explanation:

Human decides:

* what features to use

---

### 🔥 Problem:

* Requires expertise
* Not flexible

---

## 📌 10. Deep Learning Approach

### 🔹 Key Difference:

> Everything is learned **end-to-end**

---

### 🧠 Pipeline:

```id="p1v0v5"
Image → Features → Features → Features → Output
```

---

### 🔹 Layers:

1. Low-level → edges
2. Mid-level → shapes
3. High-level → objects

---

### 🔥 Key Insight:

> Features are **not manually designed**
> They are **learned automatically**

---

## 📌 11. Why Deep Learning Works Better

### Because:

* Learns optimal features
* Adapts to data
* End-to-end optimization

---

### 🧠 Meaning:

Instead of:

* choosing features manually

Model learns:

* what features work best

---

## 📌 12. Limitation of Deep Learning

### 🔹 Problem:

> We lose control

---

### 🧠 Explanation:

* We don’t know what features are learned
* Hard to interpret

---

### 🔥 Trade-off:

* More power
* Less interpretability

---

## 📌 13. Important Insight (VERY IMPORTANT)

Instructor gives real-world insight:

### Example:

Motion blur in road images:

* mostly horizontal

---

### Idea:

Instead of:

* random filters

You can:

* design filters smarter

---

### 🔥 Insight:

> Combine **domain knowledge + deep learning**

---

## 📌 14. Deep Learning is NOT Magic

### Reality:

* Needs data
* Needs tuning
* Needs understanding

---

### 🧠 Key Quote Meaning:

> You still need to think

---

# 📌 8. Introduction to Perceptron (FIRST NEURAL MODEL)

## 🔹 What is a Perceptron?

> Simplest artificial neuron

---

## 📌 9. Structure of a Perceptron

### Inputs:

* ( x_1, x_2, x_3, ..., x_n )

### Weights:

* ( w_1, w_2, ..., w_n )

---

### Operation:

[
\text{Sum} = \sum w_i x_i
]

---

# 📌 1. The Problem: XOR (Recap + Deeper)

### XOR Truth Table:

| X1 | X2 | Y |
| -- | -- | - |
| 0  | 0  | 0 |

| 1  | 0  | 1 |
| 1  | 1  | 0 |

---

## 🧠 Key Observation:

* (0,0) & (1,1) → class 0
* (0,1) & (1,0) → class 1

---

## ❌ Problem:

> No single line can separate these points

---

### 🔥 Conclusion:

* XOR is **not linearly separable**
* Single perceptron **fails** 

---

# 📌 2. Mathematical Proof of Failure

Conditions required:

* ( w_2 \ge \theta )
* ( w_1 \ge \theta )
* ( w_1 + w_2 < \theta )

---

### ❌ Contradiction:

* If ( w_1 ≥ θ ) and ( w_2 ≥ θ )
* Then ( w_1 + w_2 ≥ θ )

BUT we need:

* ( w_1 + w_2 < θ )

---

### 🔥 Conclusion:

> No solution exists → perceptron cannot solve XOR

# 🧾 Week 1 – Video 8

# 🔍 Perceptron Learning Algorithm (PLA) + Universal Approximation (DETAILED)

---

# 📌 1. The Big Question

So far:

* We manually chose weights (w₁, w₂, θ)

### ❓ Real question:

> How does the model **learn these automatically?**

---

## 📌 2. Perceptron Learning Algorithm (PLA)

### 🔹 Goal:

> Find weights **W** that correctly classify data

---

### 🔹 Given:

* Dataset with labels:

  * Positive → class 1
  * Negative → class 0

---

### 🔹 Representation:

* Input: ( X = [x_0, x_1, ..., x_n] )
* Weights: ( W = [w_0, w_1, ..., w_n] )

---

### 🔹 Condition:

[
W^T X \ge 0 \rightarrow \text{class 1}
]
[
W^T X < 0 \rightarrow \text{class 0}
]

---

# 📌 3. Algorithm Steps

### Step 1: Initialize weights randomly

---

### Step 2: Repeat until convergence

Pick a random training example:

---

### Case 1: Positive example misclassified

If:
[
W^T X < 0
]

👉 Update:
[
W = W + X
]

---

### Case 2: Negative example misclassified

If:
[
W^T X \ge 0
]

👉 Update:
[
W = W - X
]

---

### 🔥 Key Idea:

> Move decision boundary in correct direction

---

# 📌 4. Intuition (VERY IMPORTANT)

## 🧠 Think geometrically:

* W defines a **line (decision boundary)**
* We want:

  * positives on one side
  * negatives on other

---

## 📌 Dot Product Insight

[
W^T X = |W||X| \cos(\theta)
]

---

### Meaning:

| Condition     | Interpretation |
| ------------- | -------------- |
| ( W^T X > 0 ) | angle < 90°    |
| ( W^T X < 0 ) | angle > 90°    |
| ( W^T X = 0 ) | boundary       |

---

## 🧠 Intuition:

* Positive examples → want angle < 90°
* Negative examples → want angle > 90°

---

# 📌 5. Why Update Rule Works

### Case: Positive sample misclassified

We do:
[
W_{new} = W + X
]

---

### Effect:

[
W_{new}^T X = W^T X + X^T X
]

---

### Key:

* ( X^T X ≥ 0 )

👉 So:

* Value increases
* Moves toward correct classification 

---

---

### Case: Negative sample misclassified

We do:
[
W_{new} = W - X
]

---

### Effect:

[
W_{new}^T X = W^T X - X^T X
]

---

👉 Value decreases → correct classification

---

### 🔥 Final Intuition:

> PLA rotates the decision boundary until classification is correct

---

# 📌 6. Convergence Property

### Important:

> PLA converges **only if data is linearly separable** 

---

### ❌ If not separable:

* Algorithm may never stop

---

# 📌 7. Connection to Previous Lecture

* Perceptron → linear model
* PLA → learns weights for it

---

### ❗ Limitation:

Still cannot solve XOR
→ because XOR is not linearly separable

---

# 📌 8. Universal Approximation Theorem (VERY IMPORTANT)

This is a **huge theoretical result**.

---

### Statement:

> A neural network with:

* 1 hidden layer
* enough neurons

Can approximate:

> ANY continuous function

---

### 🔹 Meaning:

* Doesn’t matter how complex function is
* Network can learn it

---

### 🧠 Mathematical idea:

We want:
[
Y = G(X)
]

But we don’t know ( G(X) )

---

### Neural network learns:

[
F(X) \approx G(X)
]

---

### 🔥 Key Insight:

[
|G(X) - F(X)| \rightarrow 0
]

👉 Can get arbitrarily close 

---

# 📌 9. Why This is Powerful

### Means:

* Neural networks are **universal function approximators**

---

### Works for:

* Classification
* Regression
* Vision tasks
* Any mapping problem
